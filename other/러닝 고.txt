- 명령어

go run <소스파일.go> => 임시 디렉토리에서 컴파일한후 실행한다. 프로그램 종료후 삭제한다

go install <소스코드 저장소 위치>@latest => 서드파티 소스를 다운받아 빌드하고 GOPATH/bin에 설치한다


2. 기본데이터 타입과 선언

선언되었지만 값이 할당되지않은 모든 변수는 기본값은 제로값을 할당한다

가독성을 위해 정수리터럴은 _아랫줄 넣는것을 허용한다. 
맨처음과 끝을 제외하고 어느곳이든 넣을수있으며 숫자값 자체에는 어떤 영향도 주지않는다

룬 리터럴은 문자를 나타내며 작은따옴표로 묶어 사용한다. 큰따옴표와 혼용할수없다. ('a')

문자열 리터럴은 raw string을 사용할수있다 `(역따옴표)로 묶으면 문자열 자체가 그대로 저장된다 (작은따옴표가 아니다 )

안녕하세요
"반갑습니다"

이 문자열은 

s1 = "안녕하세요 \n \"반갑습니다\"" //일반 문자열 표현

s2 = `안녕하세요	//raw string 표현
"반갑습니다"'

go언어에서 리터럴은 타입이 지정되어 있지않다. 개발자가 특정타입을 지정하기 전까지 타입을 강제하지않는것


byte 타입은 unit8의 별칭(alias)이다.

정수형 타입은 특별한 경우가 아니라면 int를 쓴다.

float32는 소수점 6-7자리의 정밀도, float64는 15자리의 정밀도를 가진다. 대개 float64를 기본으로 쓴다

룬타입은 int32의 별칭이다. 문자를 다룰때는 rune타입을 사용하자. 
컴파일러 입장에서는 동일한 코드지만 사용자의 의도를 명확하게 하는 타입을 사용하는것이 좋다.

go에서는 모든 타입변환을 명시적으로 해야한다. 다른언어에서 0이 아닌값을 참으로 해석하는 경우가 있으나 go에서는 허용하지않는다
go는 관용적으로 간결함보다 이해도에 더 가치를 둔다




6. 포인터

- 포인터 역참조 * 전에 포인터가 nil인지 확인해야한다. nil을 가진 포인터로 역참조를 시도하면 프로그램은 패닉을 일으킨다.
nil이 사용될 가능성이 있는 로직에서는 먼저 포인터가 nil인지 체크하자

-내장함수 New(int)는 포인터 변수를 생성하며 제로값을 가리키는 포인터생성한다. 하지만 그 포인터변수는 nil이 아니다
var p2 = new(int)
p2는 0이라는 값을 가리키는 포인터이고 nil이 아니라는 점을 명심하자
nil이 아닌 포인터를 생성해야될때 참고하자

-다른 언어들도 변수내부적으로는 포인터로 구현되어있으나 , 사용자는 잘 느끼지못한다. 추상화로 내부를 감췄기때문이다. go언어는 명시적으로 값으로 사용할지, 포인터로 사용할지에 대한 선택을 제공한다. 

6-3
go언어는 기본적으로 값에의한 호출을 사용하는 언어이다. 값은 복사되고 원본의 불변성은 보장된다

6-4
포인터를 사용할때는 주의를 기울여야한다. 데이터에 변형을 가할수있고 제어흐름을 이해하기 어렵게 만들며 가비지컬렉션에게 추가적인 작업을 준다. 함수로 구조체를 전달하여 포인터로 항목을 채우기보다는 함수내에서 구조체를 만들고 반환하는것이 더 낫다

6-5 
맵과 슬라이스는 함수 파라미터로 넘어갈때 포인터가 넘어간다. 이런 이유때문에 공융api에서 입력파라미터나 반환값으로 맵의 사용을 피해야한다
맵에 어떤 키가 있는지 명시적으로 정의하는것이 없기때문에 코드를 일일이 추적해야한다. 불변성의 관점에서도 맵에 최종적으로 어떤값이 들어가있는지 확인하는 유일한 방법은 맵이 이용된 모든 함수를 추적하는것뿐이기때문에 나쁜 선택이 된다.

go의 가비지컬렉터는 짧은시간을 자주 청소하도록하는 알고리즘을 채택했다.
포인터를 드물게 사용하도록 권장되는 이유중 하나는 바로 가비지컬렉터와 관련이 있다. 
가능한한 많이 스택에 저장하여 가비지컬렉터의 작업량을 줄이는게 좋다.



charpter 7 타입,메서드,인터페이스

메서드 선언은 함수선언과 비슷한데 추가적으로 리시버(Reciever)를 명시해야한다. 관용적으로 리시버는 타입이름의 짧은 약어를 사용한다
go에서 메서드는 오버로드되지않는다. 즉, 같은 타입의 다른메서드를 동일한 이름으로 사용할수없다. 이름을 재사용하지않는것은 go철학의 일부이다

리시버 규칙
1. 메서드가 리시버의 내용을 수정한다면 포인터 리시버를 사용한다
2. 메서드가 nil인스턴스를 처리할필요가 있다면 반드시 포인터 리시버를 사용해야한다*
3. 메서드가 리시버를 수정하지않는다면 값리시버를 사용할수있다

값타입인 지역변수를 포인터 리시버와 함께 사용하면 go는 자동으로 지역변수를 포인터 타입으로 변환한다

메서드도 함수이다.
메서드를 변수에 할당하거나 타입이 func(int)int의 파라미터로 전달할수도 있다. 이것을 메서드값이라 부른다

go는 열거형이 없다. 대신 iota를 사용하여 증가하는 값을 상수세트에 할당할수있도록 한다
const(
    field1 = iota // 0
    field2
    field3
)
